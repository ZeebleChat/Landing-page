📉 What Happens When a Server Loses Boosts
🏠 Self-Hosted Servers
Removed from the global Zeeble emoji pool — no longer usable platform-wide.
Emoji data stays on the server admin's own database — Zeeble doesn't touch it.
Server admin bears their own storage cost. Zeeble bears nothing.
Boost back up → emojis are instantly re-submitted to the public pool.
☁️ Zeeble Cloud Servers
Removed from the global pool and locked within the server.
Emoji data kept on Zeeble's servers — nothing is deleted.
Members cannot use the emojis until boosts are restored.
Boost back up → emojis unlock immediately, no re-upload needed.
💵 Revenue Streams
⚡ Personal Premium Subscriptions
Users pay monthly for personal perks — more servers, more sub-accounts, priority voice, custom beam tags, profile perks, and monthly boost credits.

Managed via Stripe recurring billing. Webhook flips premium = true on their zbeam account.
🚀 Server Boosts
Premium users receive boost credits monthly and spend them on servers they want to support.

Boosts push servers through 3 tiers, unlocking public emoji slots for the whole Zeeble platform. No boosts = emojis pulled from the global pool.
🖥️ What Zeeble Hosts vs. Self-Hosted Admins
💸 Zeeble's Infrastructure Cost
zbeam — user accounts, auth, sub-accounts, beam tags
zcloud — managed cloud chat servers (capped at 5 free / 30 premium)
zpulse — DMs, DM attachments, LiveKit voice rooms
Global emoji pool — storage + serving for public emojis from boosted servers
✅ Self-Hosted Admin's Problem
Their own message history, channels, attachments
Their own voice rooms via PhaseLink
Their own server emojis (local only until boosted public)
Their own server limits, roles, moderation — Zeeble doesn't interfere
